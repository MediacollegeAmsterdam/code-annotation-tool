## ADDED Requirements

### Requirement: Question list SHALL assess learning effectiveness
The student question list SHALL include questions that evaluate whether the tool helps students understand code concepts better.

#### Scenario: Conceptual understanding questions
- **WHEN** students use the annotation tool for learning
- **THEN** questions assess whether annotations improved their code comprehension

#### Scenario: Learning process questions
- **WHEN** students work through examples
- **THEN** questions evaluate whether the tool supports their learning workflow

### Requirement: Question list SHALL assess usability for beginners
The student question list SHALL include questions that evaluate whether the tool is intuitive and accessible for coding beginners.

#### Scenario: Ease of use questions
- **WHEN** students first use the tool
- **THEN** questions assess whether interface is intuitive and requires minimal instruction

#### Scenario: Feature discovery questions
- **WHEN** students explore the tool
- **THEN** questions evaluate whether key features are discoverable

### Requirement: Question list SHALL assess engagement
The student question list SHALL include questions that evaluate whether students find the tool engaging and motivating for learning.

#### Scenario: Motivation questions
- **WHEN** students use the tool over time
- **THEN** questions assess whether tool increases their interest in coding

### Requirement: Question list SHALL be in Dutch
All questions SHALL be written in clear Dutch appropriate for students learning to code.

#### Scenario: Language appropriateness
- **WHEN** students read the questions
- **THEN** language is clear, simple Dutch without unnecessary technical terms

### Requirement: Question list SHALL gather specific feedback
The student question list SHALL include questions that elicit specific, actionable feedback rather than general impressions.

#### Scenario: Specific scenario questions
- **WHEN** asking about tool usage
- **THEN** questions reference specific features or workflows for concrete feedback

### Requirement: Question list SHALL cover workflow integration
The student question list SHALL include questions about how the tool fits into students' existing learning and coding workflows.

#### Scenario: Integration questions
- **WHEN** students use multiple learning tools
- **THEN** questions assess how this tool complements or conflicts with their other resources

#### Scenario: Practice questions
- **WHEN** students practice coding
- **THEN** questions evaluate whether tool supports hands-on practice effectively
